二、Docker环境准备

安装以及配置Docker

  1. Ubuntu下安装 Docker、将 Docker 权限给到本用户、使用中国镜像源、并开启 Docker守护进程并运行(后续需要重启才能生效)， 可以使用如下命令。
    (sudo groupadd docker || true) &&
sudo usermod -aG docker $USER &&
curl -fsSL https://get.docker.com | bash -s docker --mirror Aliyun &&
sudo docker version &&
sudo systemctl enable docker

需要先用sudo apt install url命令安装curl工具。
这个命令是用来在 Ubuntu 上安装 Docker 的，具体含义如下：

  1. 创建名为 docker 的用户组，如果该用户组已经存在则不会执行该命令：(sudo groupadd docker || true) &&
  2. 将当前用户添加到 docker 用户组中，以便能够执行 Docker 命令：sudo usermod -aG docker $USER &&
  3. 从阿里云的 Docker 官方安装脚本中下载并安装 Docker，同时指定使用 Aliyun 作为镜像源：curl -fsSL https://get.docker.com | bash -s docker --mirror Aliyun &&
  4. 验证 Docker 是否成功安装，并输出版本信息：sudo docker version &&
  5. 将 Docker 服务设置为开机自启：sudo systemctl enable docker

这些命令可以在一行中执行，用于方便快捷地安装 Docker 并配置相关设置。

  1. 查看是否安装成功

实例：hello-world

  1. 获取image文件
    docker image pull hello-world
docker image ls # 查看是否获取成功
  2. 运行image文件，生成一个正在运行的容器实例
    docker container run hello-world

实例：Ubuntu

  1. 获取image文件
    docker image pull ubuntu
docker image ls # 查看是否获取成功
  2. 运行image文件
    docker run -it ubuntu bash #其中`-i `表示以交互式的方式运行容器，`-t `表示为容器分配一个伪终端。
    -it参数组合可以让用户在终端上直接与 Docker 容器进行交互式会话，例如在 Ubuntu 容器中打开一个 Bash 终端，以便用户可以像在本地命令行一样操作容器内的文件和进程等。
  3. 列出本机正在运行的容器
    docker container ls
  4. 列出本机所有容器，包括终止运行的容器
    docker container ls --all
  5. 终止容器运行
    docker container kill [containID]
  6. 终止运行的容器文件，依然会占据硬盘空间，可以使用docker container rm命令删除。
    docker container rm [containerID]

实例：制作自己的 Docker 容器

  1. 准备项目。
    • 已有Python 3.8.10
    sudo apt update # 更新软件包列表
sudo apt install python3-pip # 安装 Python3 pip
pip3 --version # 验证 pip 是否安装成功
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple # 更换为清华源
export FLASK_APP=app
export FLASK_DEBUG=True
flask run --host=0.0.0.0

    # 项目仅有hello.py，具体内容如下
from flask import Flask

app = Flask(__name__)

@app.route("/")
def hello_world():
    return "<p>Hello, World!</p>"
  2. 在项目的根目录下新建并编写 Dockerfile 文件。
    # 使用官方 Python 运行时作为父镜像
FROM python:3.9-slim
# 将工作目录设置为 /app
WORKDIR /app
# 将当前目录下的文件都复制到 /app 中
COPY . /app
# 安装 Flask
RUN pip install Flask
# 设置环境变量
ENV FLASK_APP=hello.py
# 暴露端口 5000
EXPOSE 5000
# 运行 Flask 应用
CMD ["flask", "run", "--host=0.0.0.0"]
  3. 创建image文件
    docker image build -t hello_word .
docker image ls
  4. 测试创建的image文件
    docker run -it -p 5000:5000 --name=hello hello_word
docker container run --rm -p 5000:5000 -it hello_word /bin/bash # 便于调试，按下 Ctrl + d （或者输入 exit）退出容器

新建一个Dockerhub 账号，然后试着将自定义构建的镜像发布到 Dockerhub 上.

  1. 注册一个Dockerhub账号
  2. 为你的账号创建一个镜像仓库
  3. 在你的终端上登录dockerhub账号
    docker login -u lunrenyi
  4. 为你将要上传的镜像带上tag
    docker tag hello_word:latest lunrenyi/hello_word:v2
  5. 将镜像文件推送到你的镜像仓库中
    docker push lunrenyi/hello_word:v2

  6. 拉取镜像仓库中的文件，并进行部署测试
    docker pull lunrenyi/hello_word:v2
docker run -it -p 5000:5000 --name=hello lunrenyi/hello_word:v2

三、Git / GitHub Action 环境准备

  1. 在GitHub上新建一个仓库test_docker(https://github.com/lunrenyi/test_docker.git)
  2. 克隆到本地
  3. 编写bash脚本，并进行测试
    • bash脚本是一种文本文件，其中包含一系列的命令和参数，可以在Unix/Linux操作系统中运行。
    • 可以用来自动化和简化命令行操作和任务。
    • bash脚本的文件名后缀通常为".sh"
    #!/bin/bash
# 定义镜像名称
IMAGE_NAME="hello_world"
# 定义 Dockerfile 路径
DOCKERFILE="./dockerfile"
# 构建镜像
echo "Building image..."
docker image build -t "$IMAGE_NAME" "$DOCKERFILE"

# 运行容器
echo "Starting container..."
docker run -p 5000:5000 --name=hello "$IMAGE_NAME"

echo "Done."
  4. 上传至远程仓库
  5. 写一个简易的 GitHub Action Workflow ，并运行起来。
    # docker-image.yml
# 工作流名称
name: Docker Image CI
# 定义触发事件
on:
push:
   branches: [ "main" ]
pull_request:
   branches: [ "main" ]
# 任务
jobs:
# 构建docker镜像
build:
   # 选择运行环境
   runs-on: ubuntu-latest
   # 步骤
   steps:
   # 引入源码
   - uses: actions/checkout@v2.5.0
   - name: Build the Docker image
   run: |
       echo "hello word!"
       cat hello.py
       # 定义镜像名称
       IMAGE_NAME="hello_world"
       # 定义 Dockerfile 路径
       DOCKERFILE="."
       # 构建镜像
       echo "Building image..."
       docker image build -t "$IMAGE_NAME" "$DOCKERFILE"
       # 完成构建
       echo "Done."
